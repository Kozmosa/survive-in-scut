维护路线图 ​

最后更新: 2026-07-24 基于 TODO 汇总 的 32 个事项规划

本路线图将项目当前所有的 TODO 按优先级和内容依赖分为 7 个阶段，每个阶段包含一组可独立推进的工作项。

阶段总览 ​

阶段名称优先级TODO 数预计工作量

阶段 1 — 基础运营治理 (P0) ✅ ​

4 个 TODO → 已完成 2 项，2 项（App 能力说明、课表导入 Issue）移入独立 APP 项目维护。

先立规矩，后做事。这些工作为所有后续内容贡献提供规范和流程保障。

1.1 文档编撰规范 ​

  • 源文件: docs/others/contributing.md (第 174 行)
  • 任务: 补充附件命名规则、来源标注格式、外链规范、时间敏感信息标记约定、中英翻译同步规则、引用官方信息的标准写法
  • 依赖: 无
  • 验收标准: contributing.md 中有独立的"文档规范"章节，贡献者可以查阅

1.2 年度更新流程 ✅ ​

  • 源文件: docs/others/contributing.md (第 176 行)
  • 任务: 建立年度更新机制，明确迎新指南、校巴时刻表、校园地图、转专业政策、校医院信息、周边商家等高时效页面每学年至少核对一次的流程
  • 依赖: 1.1（规范中的时间标记约定）
  • 验收标准: 有文档化的核对表

阶段 2 — 新生刚需内容 (P0) ​

新生入学前后最需要的内容，时间敏感度最高。

2.1 迎新指南 ​

  • 源文件: docs/get-started.md (第 99 行)
  • 任务: 重写 2025 版本的迎新指南（应与最新学年信息同步）
  • 依赖: 1.1（时间敏感标记规则）
  • 参考: 现有 get-started.md 结构基本完好，需更新年份信息、时效内容
  • 验收标准: 指南涵盖注册流程、校园卡、宿舍安排、选课、军训等信息且标注了有效年份

2.2 校园安全与心理急救 ​

  • 源文件: docs/health/alive_first.md (第 11 行)
  • 任务: 补充日常安全（交通、防诈骗）、心理健康支持、灾害应急指南
  • 依赖: 无
  • 验收标准: 涵盖至少三类安全场景的应对指南

2.3 校医院就诊体验与医生推荐 ​

  • 源文件: docs/health/medical_care.md (第 106 行)
  • 任务: 补充全科、口腔科、皮肤科、中医科的就诊体验与医生推荐信息
  • 依赖: 无
  • 验收标准: 四个科室各有一段就诊体验描述

2.4 急诊经验 ​

  • 源文件: docs/health/medical_care.md (第 110 行)
  • 任务: 补充急诊流程，包括夜间突发疾病直赴校外三甲医院的报销与补开转诊单流程
  • 依赖: 2.3（同页面，可合并）
  • 验收标准: 有完整的急诊就医步骤说明

协作指引 ​

  1. 认领方式: 在对应 Issue 下留言或直接提交 PR
  2. 分支策略: 每个阶段或每个 TODO 一个独立分支，PR 标题含阶段编号（如 [Phase 2] 重写迎新指南）
  3. 格式标准: 遵循阶段 1.1 产出的文档编撰规范
  4. 翻译同步: 中文内容定稿后，应立即或在一周内更新对应英文翻译
  5. 合并前检查: 确保 npm run docs:build 通过，无 TODO 新增（除非有意识留在下一阶段）
